SQL · Commerce Mars 2025
Analyse des performances de ventes
SQL Tableau Kaggle Data cleaning
Contexte & objectif

Un dataset Kaggle de commerce en ligne contenant 100 000+ transactions sur 2 ans. L'objectif était d'identifier les segments clients les plus rentables, les produits à forte marge et les tendances saisonnières pour optimiser les campagnes marketing.

Résultats clés
+23%Revenu top segment
5Segments identifiés
100k+Transactions analysées
Q4Pic saisonnier
Méthodologie

1. Nettoyage des données en SQL (doublons, valeurs nulles, normalisation des dates).

2. Segmentation RFM (Récence, Fréquence, Montant) via des CTEs imbriquées.

3. Analyse des tendances mensuelles et par catégorie de produit.

4. Visualisation dans Tableau avec dashboard interactif filtrable.

Exemple de requête SQL
-- Segmentation RFM des clients
WITH rfm AS (
  SELECT
    customer_id,
    MAX(order_date)       AS last_order,
    COUNT(*)             AS frequency,
    SUM(total_amount)    AS monetary
  FROM orders
  WHERE order_date >= DATE_SUB(NOW(), INTERVAL 12 MONTH)
  GROUP BY customer_id
)
SELECT
  customer_id,
  DATEDIFF(NOW(), last_order) AS recency_days,
  frequency,
  ROUND(monetary, 2)         AS total_spend,
  CASE
    WHEN frequency >= 10 AND monetary >= 500 THEN 'Champions'
    WHEN frequency >= 5  AND monetary >= 200 THEN 'Fidèles'
    WHEN recency_days  <= 30              THEN 'Récents'
    ELSE 'À risque'
  END AS segment
FROM rfm
ORDER BY monetary DESC;

SQL · RH Janvier 2025
Dashboard RH & turnover
MySQL Power BI Excel
Contexte & objectif

Analyse des données RH d'une entreprise fictive (dataset IBM HR Analytics) pour identifier les facteurs de départ volontaire. L'objectif : donner aux équipes RH un outil de prédiction du risque de turnover par département et profil d'employé.

Résultats clés
16%Taux turnover global
3Facteurs principaux
1 470Employés analysés
Méthodologie

Nettoyage et exploration en MySQL, puis visualisation dans Power BI. Identification des corrélations entre turnover et variables clés : ancienneté, distance domicile-travail, satisfaction, revenus. Création d'un score de risque par département.

Exemple de requête SQL
-- Taux de turnover par département et tranche d'âge
SELECT
  department,
  CASE
    WHEN age < 30              THEN 'Moins de 30 ans'
    WHEN age BETWEEN 30 AND 45 THEN '30 à 45 ans'
    ELSE                          'Plus de 45 ans'
  END                            AS age_group,
  COUNT(*)                       AS total,
  SUM(CASE WHEN attrition = 'Yes'
            THEN 1 ELSE 0 END) AS left_count,
  ROUND(100.0 *
    SUM(CASE WHEN attrition = 'Yes'
              THEN 1 ELSE 0 END) / COUNT(*), 1
  )                              AS turnover_pct
FROM hr_data
GROUP BY department, age_group
ORDER BY turnover_pct DESC;

SQL · Finance Novembre 2024
Analyse de cohortes clients
PostgreSQL Google Sheets
Contexte & objectif

Étude du comportement d'achat par cohorte mensuelle sur un dataset e-commerce. Objectif : mesurer le taux de rétention et le LTV (lifetime value) mois par mois pour orienter la stratégie d'acquisition et de fidélisation client.

Résultats clés
34%Rétention M+1
12Cohortes analysées
$87LTV moyen à 6 mois
Méthodologie

Construction de la matrice de cohortes en PostgreSQL pur avec CTEs, puis export vers Google Sheets pour la visualisation en heatmap conditionnelle. Calcul du taux de rétention relatif et absolu, identification des cohortes à forte valeur.

Exemple de requête SQL
-- Matrice de rétention par cohorte mensuelle
WITH first_order AS (
  SELECT
    customer_id,
    DATE_TRUNC('month', MIN(order_date)) AS cohort_month
  FROM orders
  GROUP BY customer_id
),
activity AS (
  SELECT
    o.customer_id,
    f.cohort_month,
    DATE_TRUNC('month', o.order_date) AS order_month
  FROM orders o
  JOIN first_order f USING (customer_id)
)
SELECT
  cohort_month,
  EXTRACT('month' FROM AGE(order_month, cohort_month)) AS month_number,
  COUNT(DISTINCT customer_id)                             AS customers
FROM activity
GROUP BY 1, 2
ORDER BY 1, 2;
